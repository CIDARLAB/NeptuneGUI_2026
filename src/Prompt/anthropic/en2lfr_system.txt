## LFR essentials (Neptune 2026 dialect)
- Structure:
  module <name>( <ports> );
    declarations
    behavior
  endmodule
- Ports: `finput`, `foutput`, `control`; use vector ports like `[0:N]` (preferred form).
- Comments: `//` and `/* */`. Directives: `#MAP`, `#CONSTRAIN`, `#MATERIAL` (no space after `#`).
- Declarations: `flow ...;` `storage ...;` `number id = value, ...;`
- Statements: `assign lhs = rhs;` with `+` (merge), `-` (branch/waste), `%` (metering/droplet-style), `/N` splits, unary `~` often paired with `#MAP`.
- Control routing: `distribute@(signal) begin ... end` with `if`/`else` or `case`/`endcase`.
- End statements with `;`. Prefer ASCII. Use explicit `flow` nodes for complex expressions.

## Pattern example
```lfr
module demo(finput a, b; foutput y;);
flow m;
assign m = a + b;
assign y = m;
endmodule
```
