## LFR 要点（Neptune 2026 方言）
- 结构：module 名(端口); ... endmodule
- 端口：finput、foutput、control；位宽向量优先写 [0:N]。
- 注释用 // 与 /* */；指令 #MAP、#CONSTRAIN、#MATERIAL（# 后无空格）。
- 声明：flow ...; storage ...; number id = 常量, ...;
- 语句：assign 左 = 右; 运算符 + 汇合、- 分支/废液、% 计量/液滴风格、/N 分流可与 {a,b}=s/2; 配合，一元 ~ 常与 #MAP 映射算子。
- 控制：distribute@(控制量) begin ... if/else 或 case ... endcase end
- 语句以分号结束；尽量 ASCII；复杂网络多用显式 flow 中间节点。

## 与新增 LiteratureBenchmarks 对齐的规则（重要）
- 模块接口与需求严格一致：finput/foutput/control 的数量和位宽不要丢失。
- 不能漏掉出口：若一个内部流或 storage 驱动多个 foutput，必须逐个 `assign`。
- 对“门控加载/阶段加载”语义，使用 `distribute@(ctrl)`，并在分支内写 `if (ctrl == 1'b1) storage_node <= stream;`。
- 对 N 位路由/解复用语义，优先写：
  - `flow [0:(2^N-1)] lanes;`
  - `distribute@(route)` + `case({route[0], ...})` 覆盖所有状态
  - 每个 lane 显式映射到一个输出端口
- 多阶段流程尽量使用显式中间 `flow` 节点（如 feed_stage_a、merged、reaction_input）。
- `<=` 仅在 `distribute` 分支赋值中使用；普通连续赋值使用 `assign ... = ...;`。

endmodule
