# 本地上传代码说明（Git）

这份说明用于你以后在本地修改代码后，上传到远程仓库。

## 0. 进入项目目录

```bash
cd /home/cidar-ron/Downloads/NeptuneGUI_2026
```

## 1. 查看当前改动

```bash
git status
git diff
```

## 2. 更新远程信息（推荐）

```bash
git fetch origin
```

## 3. 切换到你的开发分支（如果还没建分支）

### 已有分支
```bash
git checkout 你的分支名
```

### 新建分支（示例）
```bash
git checkout -b feature/你的功能名
```

## 4. 暂存要提交的文件

### 暂存全部改动
```bash
git add .
```

### 或者只暂存指定文件
```bash
git add src/xxx.vue server/index.js
```

## 5. 提交代码

```bash
git commit -m "feat: 简要描述本次修改"
```

常见前缀建议：
- `feat:` 新功能
- `fix:` 修复问题
- `refactor:` 重构
- `docs:` 文档修改
- `chore:` 杂项维护

## 6. 推送到远程

### 第一次推送当前分支
```bash
git push -u origin 你的分支名
```

### 后续推送
```bash
git push
```

## 7. 发起合并（两种方式）

### 方式 A：在 GitHub 网页上创建 Pull Request
推送后打开仓库页面，按提示创建 PR。

### 方式 B：命令行（安装 gh 后）
```bash
gh pr create --base main --head 你的分支名 --fill
```

---

## 常见问题

### 1) 推送被拒绝（non-fast-forward）
先同步远程再推送：
```bash
git pull --rebase origin 你的分支名
git push
```

### 2) 提交了不该提交的文件
在提交前撤销暂存：
```bash
git restore --staged 文件路径
```

### 3) 想看最近提交记录
```bash
git log --oneline -n 10
```
